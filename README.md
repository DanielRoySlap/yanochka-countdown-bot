# Бот обратного отсчёта до приезда Яночки 💕

Telegram-бот, который каждый день в **00:05 по Москве** публикует в канал картинку и сообщение:

> Осталось X дней до приезда Яночки 💕

---

## Пошаговая настройка

### Шаг 1. Разобраться, что дал BotFather

Когда ты создал бота через [@BotFather](https://t.me/BotFather), он прислал сообщение примерно такое:

```
Done! Congratulations on your new bot.
You will find it at t.me/YourBotName_bot

Use this token to access the HTTP API:
1234567890:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw

For a description of the Bot API, see this page: ...
```

**Что здесь что:**

| Что написано | Что это | Нужно ли нам |
|---|---|---|
| `t.me/YourBotName_bot` | Ссылка на бота (username) | Нет, просто чтобы открыть бота |
| `1234567890:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw` | **BOT_TOKEN** — пароль бота | **Да, это главное** |
| Остальное | Подсказки BotFather | Можно не читать |

**Как выглядит BOT_TOKEN:**
- Две части через двоеточие: `цифры:буквы_и_цифры`
- Пример: `7830226105:AAFktZfvcm3NlN1R0zBAG6wtOVbIch02TvA`
- Длина обычно 40–50 символов
- **Никому не показывай** — с ним можно управлять ботом

Если потерял токен — напиши BotFather команду `/mybots` → выбери бота → **API Token** → скопируй снова.

---

### Шаг 2. Создать канал (если ещё нет)

1. В Telegram: **Новый канал** → придумай название
2. Канал может быть **публичным** (с @username) или **приватным** — оба варианта работают

---

### Шаг 3. Добавить бота в канал как администратора

1. Открой канал → **Управление каналом** (или название канала сверху)
2. **Администраторы** → **Добавить администратора**
3. Найди своего бота по username (например `@YourBotName_bot`)
4. Включи право **«Публикация сообщений»** (Post messages) — остальное можно выключить
5. Сохрани

Без этого бот не сможет постить в канал.

---

### Шаг 4. Узнать CHAT_ID канала

**CHAT_ID** — это внутренний номер чата/канала в Telegram. Выглядит так:

| Тип канала | Как выглядит CHAT_ID |
|---|---|
| Публичный канал | `-1001234567890` (начинается с `-100`) |
| Приватный канал | `-1001234567890` (тоже с `-100`) |

**Способ 1 — через @RawDataBot (проще всего):**

1. Перешли любое сообщение из своего канала боту [@RawDataBot](https://t.me/RawDataBot)
2. Он ответит JSON — найди строку `"chat":{"id":-1001234567890`
3. Число `-1001234567890` — это твой **CHAT_ID**

**Способ 2 — для публичного канала:**

Можно использовать `@channelusername` (с @), например `@my_yanochka_channel` — но надёжнее всё равно взять числовой ID через RawDataBot.

**Важно:** CHAT_ID всегда **отрицательное** число, для каналов обычно начинается с `-100`.

---

### Шаг 5. Добавить секреты в GitHub

Репозиторий уже на GitHub. Осталось добавить два секрета:

1. Открой репозиторий на GitHub
2. **Settings** → слева **Secrets and variables** → **Actions**
3. Нажми **New repository secret**

Добавь **два** секрета:

| Имя секрета | Что вставить | Пример |
|---|---|---|
| `BOT_TOKEN` | Токен от BotFather (часть с двоеточием) | `1234567890:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw` |
| `CHAT_ID` | ID канала (с минусом!) | `-1001234567890` |

Имена должны быть **точно** `BOT_TOKEN` и `CHAT_ID` — как в таблице, без пробелов.

---

### Шаг 6. Проверить, что всё работает

1. На GitHub открой вкладку **Actions**
2. Слева выбери **Daily Countdown**
3. Справа **Run workflow** → зелёная кнопка **Run workflow**
4. Через ~30 секунд в канале должно появиться сообщение с картинкой

Если ошибка — открой запуск workflow, кликни на красный крестик и посмотри лог. Частые причины:
- Неверный `BOT_TOKEN`
- Неверный `CHAT_ID` (забыл минус или перепутал канал)
- Бот не добавлен админом в канал

---

### Шаг 7. Автоматическая отправка

После настройки секретов бот будет сам писать в канал **каждый день в 00:05 по Москве**.

Первое сообщение — **4 июля 2026** в 00:05 МСК.

## Локальный тест (необязательно)

Если хочешь проверить с компьютера:

```bash
# Скопируй шаблон
copy .env.example .env

# Открой .env и вставь свои значения:
# BOT_TOKEN=1234567890:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw
# CHAT_ID=-1001234567890

pip install -r requirements.txt
python main.py
```

## Картинки

В папке `images/` лежат 67 placeholder-картинок. Замени их на свои с милыми зверятами (свинки, капибары, котики и т.д.):

- `day_01.png` — для 67 дней до приезда
- `day_67.png` — для дня приезда

Перегенерировать placeholder:

```bash
pip install -r requirements-images.txt
python generate_images.py
```
